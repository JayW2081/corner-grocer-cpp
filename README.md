# Corner Grocer Item Frequency Tracker

## Project Summary

The Corner Grocer project is a C++ console application designed to help a grocery store analyze how often items were purchased throughout the day. The program reads item data from an input file, counts the frequency of each item, and allows the user to search for a specific item, print all item frequencies, or display a histogram of the data. This solves the problem of manually reviewing sales records and gives the store a faster way to understand purchasing trends.

## What I Did Well

One thing I did particularly well was organizing the program around clear functionality. The menu gives the user simple options, and each option performs a specific task. I also made the output easy to understand by displaying the item counts clearly and using a histogram to visually represent item frequency. This makes the program more useful for someone who may not want to read through raw data.

## Where I Could Enhance the Code

I could enhance the code by adding stronger input validation and better error handling for file operations. For example, the program could provide clearer messages if the input file is missing or if the user enters an invalid menu choice. These improvements would make the program more secure and reliable because it would handle unexpected input without crashing or producing incorrect results.

Another improvement would be separating the program into multiple files, such as a header file, implementation file, and main file. This would make the code easier to maintain as the project grows.

## Challenges and How I Overcame Them

The most challenging part of this project was working with file input and storing item frequencies correctly. I had to make sure the program read each item from the file, updated the correct count, and then displayed the results accurately. I overcame this by breaking the problem into smaller steps and testing each part of the program separately.
